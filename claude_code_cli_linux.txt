# =======
# INSTALL
# =======

1. Install Node.js (includes npm):
   a. Download and setup NodeSource repository (e.g., for Ubuntu/Debian):
      curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
   b. Install Node.js:
      sudo apt install -y nodejs
   c. Verify installation:
      node -v
      npm -v
2. Install Claude Code:
   a. Install globally:
      sudo npm install -g @anthropic-ai/claude-code
   b. Verify installation:
      claude --version
   c. (Optional) Diagnose the installation:
      claude doctor


# ===========
# QUICK GUIDE
# ===========

1. Login:
   Run:
      claude
   - On first launch, run /login (or follow the prompt)
   - Follow the browser authentication flow
   - Sign in with your Claude (Anthropic) account, or use an API key
2. Ask a question:
   a. Interactive mode:
      claude
      > your question here
   b. One-line mode (print and exit):
      claude -p "your question here"
3. Use files as context:
   a. Option A (recommended - piping):
      cat FILE_PATH | claude -p "analyze this code"
   b. Option B (reference in prompt with @):
      claude "analyze the following code: @FILE_PATH"
4. Help:
      claude --help

# ==========================
# SESSION & WORKSPACE FOCUS
# ==========================

The directory where you launch 'claude' (Session Directory) defines your project:

1. Scope: History and auto-saves are scoped to the launch directory.
2. Discovery: Claude Code searches for CLAUDE.md files and local .claude/
   configurations starting from this directory upward.
3. Data: Project-specific data and history are stored under ~/.claude/
   keyed to the launch directory path.


# =========
# ABILITIES
# =========

1. Provides terminal access to cloud-based Claude LLM via a CLI interface
2. Supports:
   - Question answering
   - Code generation
   - Code review and debugging
   - Text summarization
   - Iterative conversational workflows
3. File system interaction:
   - File reading for context
   - Code editing (with user confirmation / permission prompts)
   - Running shell commands (with permission)

# ==================
# USEFUL CLI TAGS (/)
# ==================

/help           - Show available commands and usage help
/model          - View or switch between available models
/compact        - Summarize and compress context to save tokens
/clear          - Clear the current conversation history
/resume         - Browse and resume a previous conversation
/memory         - Load and edit CLAUDE.md files
/init           - Initialize a project (generates a tailored CLAUDE.md)
/add-dir        - Manage session scope (add directories to the workspace)
/config         - View or change settings
/login          - Sign in / switch account
/cost           - Show token usage and cost for the current session
/exit           - Exit the CLI session


# ==========
# MCP SERVERS
# ==========

Claude Code extends its abilities through MCP (Model Context Protocol) servers.

1. Add an MCP server (example - generic):
   a. From the terminal:
      claude mcp add <name> -- <command to launch the server>
2. Verify / manage MCP servers:
   a. From the terminal:
      claude mcp list
   b. Within an interactive session:
      /mcp
3. Safety:
   MCP servers run locally on your machine and operate via the Model Context
   Protocol. Data is handled on your hardware and only accessed with your
   explicit permission.
4. Features:
   - Connect external tools and data sources (e.g. Google Workspace).
   - Enables interaction with services such as Google Calendar and Gmail.
   - Allows searching emails, managing schedules, and creating events directly
     from the terminal.


# ==============================
# PROJECT PERSISTENCE (CLAUDE.md)
# ==============================

CLAUDE.md files provide persistent context and rules for the AI (coding standards,
project goals, etc.) without repeating them in every prompt.
Context is loaded hierarchically:
1. Global:    ~/.claude/CLAUDE.md (applies to all sessions)
2. Workspace: CLAUDE.md in project root or parent directories
Use /memory to view and edit the active CLAUDE.md files. Changes are picked up
on the next prompt.
