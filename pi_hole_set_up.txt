# ===============
# PI HOLE CONCEPT
# ===============

1. PI hole is a DNS server that blocks ads and trackers
2. When a device on your network tries to access a website, it first queries the Pi hole for the IP address of the website
3. The Pi hole checks if the website is in its blocklist and if so, it blocks the request
4. If the website is not in its blocklist, it forwards the request to the upstream DNS server and returns the IP address to the device

# ===============
# STATIC IP SETUP
# ===============

1. Before installing Pi-hole, its recomanded to assign a static IP to your Raspberry Pi so its address doesn't change on reboots.
2. Find your current connection name: nmcli connection show --active
3. Set a static IP (In this example 10.0.0.1, 24 marks subnet bits) and gateway (In this example 192.168.1.1) on your active connection:
   -> nmcli connection modify "<CONNECTION NAME>" ipv4.addresses 10.0.0.1/24 ipv4.gateway 192.168.1.1 ipv4.method manual
4. Restart the connection to apply: 
   -> nmcli connection down "<CONNECTION NAME>" && nmcli connection up "<CONNECTION NAME>"

# ===============================
# PI HOLE INSTALLATION AND CONFIG
# ===============================

1. On the linux machine run: curl -sSL https://install.pi-hole.net | bash
2. Follow the instructions on the screen:
   -> Choose the correct interface (eth0, wlan0, etc) that provides internet to the pi clients.
   -> Select your preferred Upstream DNS Provider (Google).
   -> Note down the Admin Webpage login password shown at the very end of the installation!
3. To uninstall use: pihole uninstall, later on remove etc/pihole and etc/.pihole 

# ==============================
# PI HOLE MANAGEMENT & WEBPAGE
# ==============================

1. Access the Pi-hole admin dashboard from any browser in your network: http://<PI HOLE IP>/admin
2. Change or reset your web password anytime via terminal: sudo pihole setpassword
3. To disable Pi-hole blocking: pihole disable
4. To re-enable Pi-hole blocking: pihole enable
5. To update Pi-hole to the latest version: pihole -up
6. After adding block lists, use: sudo pihole -g, site with domains collaction to block: https://firebog.net/

# ========================
# CLIENT DNS CONFIGURATION
# ========================

1. Open the Wi-Fi settings on your client device (Laptop, iPad, etc.).
2. Select the network serviced by the Pi-hole and open its settings.
3. Change the IPv4 DNS configuration from "Automatic" (DHCP) to "Manual".
4. Enter the Pi-hole's static IP address as the DNS server.
5. To test DNS source you can use "nslookup <Domain>" and see what is the DNS IP. You can use specific DNS by using "nslookup <Domain> <DNS IP>" 

# ===============================
# ROUTER DNS & PORT CONFIGURATION
# ===============================

1. Listening Interface: Pi-hole must listen on all interfaces (eth0, wlan0) to serve the Hotspot.
   -> Manual file edit: /etc/pihole/pihol.toml (Change interface="eth0" to interface="" and listeningMode="LOCAL" to listeningMode="ALL")
2. Port 53 Conflict: port 53 is used for DNS services, NM and pihole bioth creating DNS servers and thus have a conflict over this port preventing Pi-hole from starting.
3. To fix the conflict, disable NM DNS server using the following steps:
   -> Create a file such as: "sudo nano /etc/NetworkManager/conf.d/no-dns.conf" including "[main]\n dns=none"
   -> systemctl restart NetworkManager
   -> Edit "/etc/resolv.conf" and change DNS ip (A file auto generated by NM without the above configuration addition)
   -> systemctl restart NetworkManager
